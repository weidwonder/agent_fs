# 开发与回归要求

## 打包后端到端验证

用户明确要求：每次修复完成后，不能只做源码层验证，必须执行面向安装产物的端到端回归，尽量在用户再次手工复测前发现问题。

当前执行要求：

- macOS 本地安装链统一走 `./scripts/install_macos.sh`
- 安装脚本完成后必须自动执行 `scripts/verify-packaged-app.mjs`
- 烟测至少覆盖：
  - `apache-arrow` 已进入打包产物
  - `DocxConverter` / `ExcelConverter` 资源存在且可启动
  - 打包后的 JS 入口能解析并启动对应转换器
- 若烟测失败，不应向用户宣称“已修复”或“已重新打包完成”
- 修复完成后的默认交付标准是：源码测试通过、重新打包安装成功、打包后烟测通过
