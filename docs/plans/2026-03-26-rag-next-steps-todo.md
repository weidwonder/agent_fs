# RAG 后续优化 TODO

> 目的：记录当前 RAG 体系下一阶段最值得做的三项通用优化，后续按需实施。

## 1. Rerank

- 为 TopN 候选增加二次重排层，目标是提升“文件已命中但首屏 chunk 不够直接”的问题。
- 优先考虑轻量方案：
  - 基于 query / keyword / 标题锚点的规则重排
  - 本地 rerank 模型
- 评估重点：
  - 是否提升 `strict hit`
  - 是否减少“候选命中但展示不佳”

## 2. 结构化 Chunking

- 对法规、准则、指引类文档按结构切分，而不是主要依赖固定长度。
- 优先识别的结构锚点：
  - 标题
  - 章
  - 条
  - 款
  - 列表项
- 目标：
  - 降低同文件相邻章节互相抢位
  - 提高条款级定位精度

## 3. 正式评测

- 建立分层评测集，至少区分：
  - `库内明确可答`
  - `库内不可答`
- 对 `库内明确可答` 样本统计：
  - 文件级 Top1 命中率
  - 文件级 Top3 命中率
  - chunk 级 strict hit
  - 候选命中但展示不佳占比
- 对评测样本记录：
  - `preferred_gold_file`
  - `acceptable_gold_files`
  - `gold_anchor`
- 后续目标：
  - 形成可重复执行的评测脚本
  - 对改动前后做 A/B 对比
